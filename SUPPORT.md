## How to Get Support

### Bug Reports

If CyberTuz is not working as expected — crashes, broken output, missing features — open a **Bug Report** issue on GitHub:

[Open a Bug Report](https://github.com/djunekz/cybertuz/issues/new?template=bug_report.yml)

Please include:
- Your Termux version (`termux-info`)
- Your Android version
- Exact steps to reproduce
- What you expected vs. what happened
- Terminal output or error messages

### Feature Requests

If you have an idea for a new module, language, or improvement:

[Open a Feature Request](https://github.com/djunekz/cybertuz/issues/new?template=feature_request.yml)

### General Questions and Support

For questions about how to use CyberTuz, module content, or general cyber security learning guidance:

[Open a Support Request](https://github.com/djunekz/cybertuz/issues/new?template=support_request.yml)

### Security Vulnerabilities

**Do not open a public issue for security vulnerabilities.**

See [SECURITY.md](SECURITY.md) for responsible disclosure instructions.

---

## Response Times

CyberTuz is maintained by an individual contributor. Response times are not guaranteed, but we aim to:

- Acknowledge new issues within a few days
- Triage and respond to bugs within one week
- Address critical issues as quickly as possible

---

## Community Guidelines

When asking for or providing help:

- Be respectful and patient
- Provide as much detail as possible
- Do not ask for help with unauthorized access or illegal activities
- See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for full community standards
